问题描述

当前的看板系统在处理超过 500 个任务卡片时存在明显的 DOM 渲染瓶颈。为了提升大中型项目的操作流畅度，我们需要引入虚拟列表（Virtual List）技术并重写现有的拖拽交互引擎。

主要改进点：

  • 使用 Canvas 或更高效的 SVG 方案重构任务连接线。
  • 引入全局状态管理，减少不必要的组件重绘。
  • 针对 Graphite 风格优化 UI 细节，确保在极高密度下依然保持可读性。
  • 增加快捷键支持 (Ctrl + D 复制, E 编辑)。
